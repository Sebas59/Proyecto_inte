Planeación del Proyecto

Casos de Uso

Se identificaron los siguientes casos de uso principales para el sistema, detallando las interacciones entre el usuario y la aplicación:

  • Gestión de Vehículos:
    • Registrar nuevo vehículo (marca, modelo, año, tipo combustible, tamaño tanque, imagen del vehiculo).
    • Ver lista de vehículos (con filtros).
    • Editar información de vehículo.
    • Eliminar vehículo (moviéndolo a historial).
    • Restaurar vehículo desde historial.
  • Gestión de Precios de Combustible:
    • Registrar nuevo precio (ciudad, localidad, tipo combustible, precio por galón).
    • Ver lista de precios de combustible (con filtros).
    • Editar información de precio.
    • Eliminar precio (moviéndolo a historial).
    • Restaurar precio desde historial.
  • Cálculo de Costos de Combustible:
    • Asignar combustible a un vehículo.
    • Visualizar el costo de combustible por vehículo.
  • Navegación:
    • Acceder a la página de inicio.
    • Navegar a la lista de vehículos, precios de combustible, historiales y la sección "Nosotros".

Objetivos Propuestos y Fuente de los Datos

Los objetivos principales del proyecto fueron:

  • Ofrecer una solución sencilla y accesible para que los usuarios gestionen sus vehículos y el consumo de combustible.
  • Permitir la trazabilidad de vehículos y precios de combustible, incluso después de ser "eliminados" lógicamente.
  • Proporcionar una interfaz de usuario limpia e intuitiva que mejore la experiencia del usuario.
  • Demostrar el uso de FastAPI con SQLModel para un desarrollo backend robusto y eficiente.
  • Implementar un diseño frontend responsivo con Bootstrap para compatibilidad multidispositivo.

La fuente de los datos para los precios de combustible se espera que provenga de una "entrada manual del usuario", simulando la recopilación de información de diferentes ciudades y localidades. Para los datos de vehículos, la información también es "introducida directamente por el usuario según sus propios activos".
